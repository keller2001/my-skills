# 异常处理能力

本文件覆盖网页因自身设计缺陷导致的数据错误、信息重复、元素遮挡等问题。

## 常见异常类型及应对

### 1. 数据错误

**表现**：价格显示为 0 或负数、日期格式混乱、数值明显不合理

**应对策略**：
- 交叉验证：查找其他来源确认
- 逻辑判断：根据常识判断合理性
- 标注存疑：在输出中标注"待核实"
- 跳过异常：不影响核心任务时可跳过

**示例**：
- 商品价格显示 -100 元 → 标注"价格异常，待核实"
- 发布日期显示 2025-13-45 → 标注"日期格式异常，待核实"

### 2. 信息重复

**表现**：同一帖子出现多次、列表项内容完全一致

**应对策略**：
- URL 去重：相同 URL 只保留一条
- 近似去重：内容相似度超过 90% 视为重复
- 记录重复率：在报告中说明去重情况
- 保留高质量版本：选择信息更完整的版本

**示例**：
- 搜索结果中出现 3 条相同微博 → 去重后保留 1 条
- 商品列表中有重复 → 合并统计

### 3. 元素遮挡

**表现**：弹窗遮挡目标按钮、广告覆盖部分内容

**应对策略**：
- 关闭遮挡物：寻找关闭按钮（X 图标）
- 滚动避让：上下滚动避开遮挡区域
- 结构绕过：通过页面结构直接定位目标
- 人工接管：无法自动处理时请求用户协助

**示例**：
- 登录弹窗挡住所有内容 → 尝试关闭或请求用户登录
- 广告横幅挡住导航栏 → 滚动页面或使用快捷键

### 4. 加载失败/超时

**表现**：页面部分元素未加载、图片显示破损、列表无限加载

**应对策略**：
- 等待加载：给予合理等待时间（5-10 秒）
- 刷新重试：点击刷新按钮或 F5
- 降级处理：仅提取文字等可用信息
- 报告卡点：明确说明哪些内容无法获取

**示例**：
- 图片加载失败 → 仅提取文字信息并标注"图片缺失"
- 列表无限滚动 → 设定最大轮次（如 5 轮）后停止

## 异常处理通用流程

1. 发现异常
2. 分类诊断（确定异常类型）
3. 评估影响程度（高/中/低）
4. 选择处理策略（继续/降级/终止）
5. 记录异常（在报告中说明）
6. 继续执行或请求人工接管
