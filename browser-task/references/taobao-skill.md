# Taobao Skill（子 Skill）

> 本文件是 GUI 浏览器自动化任务技能的子模块，当且仅当用户明确要在淘宝/天猫完成商品检索、比价选优或加购时被 `SKILL.md` 路由到。

本 Skill 用于把“我要买/比价/选靠谱款/加购”转成可执行结果：**检索 → 选优（品牌/价格/可靠性）→ 规格确认 → 加入购物车（单个/批量）→ 清单交付**。

## 适用场景

- 帮我在淘宝/天猫搜某类商品，并给出靠谱选项与推荐理由
- 帮我对比品牌与价格后，选择更可靠的商品
- 帮我把一个或多个商品加入购物车（含不同规格/数量）
- 场景例（不限定品类）：帮我加购这些商品：A、B、C、D（可混合母婴/家电/3C/个护/食品等）
- 场景例：我预算 3000，帮我选一款可靠的扫地机器人并加购
- 场景例：同款不同店铺差价太大，帮我解释差异并选更靠谱的

## 输入（缺失则按默认执行）

- **商品清单**：每项至少包含品类名；可选：品牌/型号/规格/预算/数量/禁忌点
- **预算策略**：默认中位价优先（不选明显异常低价）；如用户要求“性价比/大牌/最低价”，按其偏好调整
- **偏好**：默认优先天猫与品牌旗舰店/官方店；可选：是否接受非旗舰/代购/海外
- **收货信息约束**：默认不强依赖地域；如页面提示不发货/运费异常，改选可发货选项
- **加购方式**：默认批量加购后给出购物车核对清单；如用户只要建议不加购，则不执行加购步骤
- **交付格式**：默认输出 1 份“加购清单 + 备选清单 + 待确认问题”

## 强制约束（Hard Requirements）

- 不得编造商品信息（价格、销量、评价数、品牌授权、店铺资质、发货地、优惠券、赠品、链接）。
- 只在已可回溯页面中读取并引用信息（商品详情页/店铺页/搜索结果页）；每个被推荐或被加购的商品必须给出可回溯链接。
- 遇到登录/验证码/访问受限，立即切换为“请求用户协助”模式：向用户索要可打开的商品链接/搜索页链接，或让用户提供截图；在未看到页面证据前，不得凭经验补全。
- 必须做“品牌与价格的对比选优”，且把对比依据落到可核验字段（店铺类型、评价摘要、参数、退换保障、价格区间、同规格单价）。
- 对每个“首选/已加购”商品，必须提供可核验的“可靠性证据链”：品牌与背书（如旗舰/官方/授权等页面标识）、口碑/评价关键信号与可见数据、保障与售后条款；每条证据都要能追溯到对应页面链接。
- 默认以“可靠性优先”而不是“最低价优先”；对明显异常低价、信息缺失、评价噪声大、店铺资质弱的候选项默认降级为备选或剔除。
- 无论是否完成加购，最终都必须交付“备选清单”，包含备选链接与备选理由（说明与首选差异点与风险点）。

## 常规操作（强制）

- 导航与搜索只使用页面可见元素（链接、筛选、排序、搜索框等），不臆测不可见功能。
- 当详情页在新标签页打开时，完成采集后通过关闭当前标签页返回列表，不依赖浏览器返回。
- 在搜索结果页优先使用品牌、销量、评价、价格区间、店铺类型等筛选/排序能力辅助筛选候选；若用户未指定品牌，优先勾选多个知名/可靠品牌而不是只选单一品牌。

## 工作流

### 0) 任务拆解（强制）

- 将用户需求拆成“要买的每个品类/商品项”，为每项定义：用途、关键参数、可接受价位、数量。
- 若用户未给关键参数，按“安全/通用/新手默认”兜底，并在输出的“待确认问题”里列出需要用户最后拍板的参数。

### 1) 生成检索式（每项最多 3 组）

对每个商品项生成互补检索式，覆盖“品牌/材质/规格/适用年龄/安全标准”：

- 基础召回：`品类 + 关键规格词`
- 品牌聚焦：`品类 + 头部品牌/用户指定品牌`
- 可靠性聚焦：`品类 + 天猫/旗舰店/官方`

扩展词生成方法（按品类套用，优先选页面可核验的关键词）：

- 规格/尺寸/容量/功率/重量/套装数
- 材质/面料/成分/工艺
- 型号/代际/适配（机型/接口/系统/年龄段/人群）
- 安全与合规（国家标准/认证/适用月龄/适用范围）
- 场景词（通勤/家用/办公/户外/旅行/新生儿等）

同义词扩展示例（仅作示范，可替换为任意品类的关键词组合）：

- 奶瓶：`PPSU / 玻璃 / 宽口 / 防胀气 / 新生儿 / 160ml 240ml`
- 婴儿降噪耳机：`婴儿 降噪 耳罩 / 防噪音 耳罩 / 0-3岁`
- 婴儿辅食：`婴儿 辅食 米粉 / 泡芙 / 面条 / 肉松`（若用户未说明月龄，先按“仅推荐合规品牌与清晰标注适用月龄”）
- 口水巾：`口水巾 纯棉 / A类 / 围兜 / 纱布`
- 扫地机器人：`扫地机器人 扫拖一体 / 自动集尘 / 避障 / 地毯增压 / 水箱`

### 2) 候选采集（每项固定 3 个候选，强制）

为保证流程稳定且避免“进详情后回不去同批备选”，按以下方式固定采集 3 个候选：

- 使用第 1 步生成的检索式进入搜索结果页，在该页优先使用筛选/排序（如品牌多选、销量、评价、店铺类型等）把候选收敛到更可靠范围，再选择“最合适的 3 件商品”作为候选（以相关性与店铺可靠性优先，不追最低价），并记录该搜索结果页链接（后续用于回到同一结果页）。
- 按候选顺序逐一打开详情页采集信息；每次进入详情页必须一次性同时收集“商品详情页链接 + 关键参数”，避免后续重复进入同一商品补采参数或补链接。
- 每看完 1 个候选后，必须强制执行：先确认已记录该商品详情页链接与关键参数 → 再调用 open_url_in_browser 打开“本次记录的搜索结果页链接”回到结果页 → 定位并进入下一个候选的详情页（重复 2 次）。
- 完成 3 个候选采集与评分后，再次通过搜索框重新输入同一条检索式回到搜索结果页，进入最终选中的“首选”商品详情页执行加购。
- 参数定位提示：多数商品的“参数/规格/详情”区域通常在详情页靠下位置，常见出现在页面右下角附近；若首屏未看到参数，持续使用滚轮向下滚动直到出现“参数/规格/宝贝详情”等入口，再进入读取关键参数。

至少采集以下字段（缺失则标注未知）：

- 商品标题（含规格/适用信息）
- 价格与到手价口径（以页面展示为准，注明是否含券/活动）
- 店铺类型（天猫/淘宝；旗舰店/专卖店/专营店/普通店）
- 品牌信息（页面可见的品牌/品牌方字段；若仅能从标题推断则标注未知，不强行写死）
- 品牌背书/资质标识（如“品牌旗舰店/官方店/授权”等页面可见标识；若可进入店铺页，补充店铺首页链接）
- 关键参数（材质/容量/适用月龄/标准/成分/尺寸等，按品类选择）
- 退换与保障要点（如“7 天无理由”、运费险、正品保障等页面可见项）
- 口碑与评价证据（只摘录页面可见的典型正/负面关键词；若页面展示了评分/评价数/店铺评分等数字信息，可记录其原值与口径，不可见则标注未知）
- 链接与抓取时间（系统当前日期）

### 2.5) 搜索回退与备选留存（强制）

淘宝/天猫场景下，从商品详情页返回同一批搜索结果不稳定，因此必须将“回退检索”视为流程的一部分，并保证备选可回访：

- 对每个商品项，明确记录本次使用的检索式（原样复用，不做临时改词），并在输出中给出该检索式。
- 采集 3 个候选的过程中，必须通过 open_url_in_browser 回到本次记录的搜索结果页链接 3 次（前两次用于回到结果页继续采集，最后一次用于回到结果页并进入首选加购）；若搜索结果页链接不可用，再退化为在搜索框重复检索同一检索式。
- 输出里必须包含：首选商品链接 + 2 个备选商品链接，并为三者都提供本次已采集的关键参数摘要，避免用户需要再次打开链接确认规格。

### 3) 去噪与初筛（强制）

剔除或降级以下候选：

- 标题/参数不清晰，关键规格缺失且详情页难以确认
- 明显异常低价且无法解释（同规格远低于主流区间）
- 店铺资质弱且无法在页面看到可靠保障信息
- 评价区集中出现“做工差/异味/漏液/尺寸不符/假货/二手翻新”等风险信号（按品类映射）
- 规格选择复杂但页面无清晰指引（容易选错）

### 4) 可靠性与性价比评分（0–5 分，强制）

对每个候选按维度评分并给 1 句话理由：

- **相关性**：是否满足用户用途与关键参数
- **可靠性**：店铺类型与背书标识、保障信息、参数透明度、口碑/评价风险信号（必须能对应到可回溯页面证据）
- **性价比**：同规格价格区间位置、是否存在更合理的同类替代
- **售后友好**：退换、运费险等页面可见条款
- **选型确定性**：规格是否清晰、是否容易买错

默认选优策略：

- 每项输出 1 个“首选” + 2 个“备选”
- 若用户明确“只要最便宜”，仍先给“可靠底线”过滤，再在通过过滤的集合里选最低价

### 5) 规格与数量决策（缺失则默认）

默认规则（可被用户覆盖）：

- **数量**：默认每项 1 件；用户给了数量则按数量
- **规格**：选择“最通用/标注明确/适用面更广”的规格，并在清单中写明
- **组合装**：优先单价合理且不浪费的装数；不为追求低单价而强行大包装

### 6) 加入购物车（单个/批量）

执行加购时，严格按“从页面读取 → 选规格 → 选数量 → 点击加入购物车 → 回到购物车核对”的顺序。

#### 单个加购

- 打开商品详情页链接
- 选择规格（颜色/尺码/容量/组合等）
- 若规格选项/按钮为灰色不可选，通常意味着该规格无货或不可用；改选其他可用规格，或切换到备选商品继续加购
- 设置数量
- 点击加入购物车
- 在输出中记录“已加购状态”与“核对点”（规格、数量、到手价口径）

#### 批量加购

- 为每个商品项选择 1 个首选商品（必要时保留备选不加购）
- 逐个完成“规格 + 数量 + 加购”
- 打开购物车进行核对：是否全部在车、是否同款不同规格混淆、是否出现预售/不发货提示、是否被自动替换规格

### 7) 交付格式（强制）

输出以下三个部分：

1) **加购清单（已加购）**：一张表，至少包含：商品项、选中商品、品牌（若可见）、店铺类型、关键规格、数量、价格口径（若可见）、可靠性证据（背书/口碑/保障要点）、证据来源链接（商品页/店铺页/搜索页）、抓取时间

2) **备选清单（未加购）**：每项 1–2 个备选，必须包含：备选商品链接、备选理由（与首选差异点：更便宜/同品牌不同店/规格更合适/风险更高等）、建议使用场景（例如“首选无货/不发货/到手价波动大时替换”）

3) **待确认问题**：仅列真正会影响下单的关键项（例如：奶瓶材质偏好、宝宝月龄/辅食禁忌、降噪耳罩尺寸与佩戴、口水巾材质/数量）

## 场景示例：批量加购 4 个品类

用户：帮我加购这 4 个商品：奶瓶、婴儿降噪耳机、婴儿辅食、口水巾

执行要点：

- 将 4 个品类拆为 4 个商品项分别检索与打分
- 奶瓶与辅食优先“品牌/店铺可靠性”；降噪耳机优先“适配年龄与佩戴舒适”；口水巾优先“材质与数量性价比”
- 每项选择 1 个首选完成加购，并在最终输出给 2 个备选与 2–4 个待确认问题（例如奶瓶材质 PPSU vs 玻璃、辅食适用月龄与过敏禁忌、口水巾厚薄与季节）
