# 操作类任务验证

本文件覆盖点赞、收藏、评论、转发、关注等需要验证状态的操作。

## 核心问题

1. 状态识别不足：看不出是否已点赞/收藏
2. 发送验证缺失：误以为评论已发送，实际未发送
3. 反向操作风险：对已点赞的内容再次点击导致取消

## 通用原则

- 操作前必查状态：避免重复操作导致反向效果
- 操作中观察反馈：即时确认操作生效
- 操作后二次验证：刷新页面或查看列表确认
- 失败时明确告知：不要假设"应该成功了"

## 各操作验证要点

提示：不同页面的"点赞、收藏、评论、转发、关注"等组件的外观、交互、表现各不相同，下面对于常见的组件进行举例说明，做到最大程度的覆盖

### 点赞操作

**检查状态**：
- 灰色/空心 → 未点赞
- 红色/实心 → 已点赞

**执行步骤**：
1. 检查当前状态（避免重复点击）
2. 点击按钮中心
3. 观察即时反馈

**验证标准**：
- 按钮变红/实心
- 点赞数 +1
- 出现"已赞"提示
- 按钮下方字符串含有"已赞"字眼

**失败处理**：
- 等待 2 秒后刷新页面
- 重新检查状态
- 如仍失败则告知用户

### 收藏操作

**检查状态**：
- "收藏" → 未收藏
- "已收藏"/灰色 → 已收藏

**执行步骤**：
1. 检查当前状态
2. 点击菜单中的"收藏"选项
3. 观察即时反馈

**验证标准**：
- 菜单变为"已收藏/取消收藏"
- 出现"收藏成功"提示

**失败处理**：
- 刷新页面后检查菜单状态
- 如仍显示"收藏"则告知用户

### 评论操作（最容易出错）

**检查状态**：无状态标识，直接输入

**执行步骤**：
1. 点击输入框聚焦
2. 输入评论内容
3. 点击发送按钮
4. 等待 3 秒

**验证标准（关键）**：
- 出现"评论成功"提示
- 评论出现在评论区顶部
- 评论数 +1
- 输入框清空

**超时处理**：
1. 等待 3 秒
2. 刷新页面
3. 检查是否真的发送成功
4. 如未成功则告知用户并询问是否重试

### 关注操作

**检查状态**：
- "关注" → 未关注
- "已关注"/灰色 → 已关注

**执行步骤**：
1. 检查当前状态
2. 点击"关注"按钮
3. 观察即时反馈

**验证标准**：
- 按钮变为"已关注"
- 按钮变灰

**失败处理**：
- 刷新页面后检查按钮状态
- 如仍显示"关注"则告知用户

### 转发操作

**检查状态**：通常无"已转发"状态标识

**执行步骤**：
1. 点击转发按钮
2. 选择转发方式（公开/私信）
3. 确认发布

**验证标准**：
- 出现"发布成功"提示
- 进入个人主页看到刚转发的内容
- 原微博转发数 +1

**失败处理**：
- 检查网络状态
- 刷新原页面查看转发数
- 如失败则告知用户
