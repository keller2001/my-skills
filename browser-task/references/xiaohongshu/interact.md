# 互动运营工作流（点赞/收藏/关注/评论/回复）

目标：在获得用户明确授权且当前环境可操作时，完成点赞、收藏、关注、评论、回复等互动动作，并回传每一步结果。

## 前置原则

- 所有账号内动作都必须基于“用户明确授权”
- 若用户未登录或当前环境不可操作：停止执行账号动作，改为输出“手动操作步骤 + 检查点”，不要臆造已执行结果
- 节奏安全：避免短时间连续高频操作；每次动作之间保持间隔并控制总量
- 目标明确：只对“明确的目标链接/当前页面可见笔记/明确的评论”操作，不要猜测或批量乱点

## 输入信息（缺什么先补什么）

- 目标笔记链接（或明确到标题+作者+页面位置）
- 要执行的动作：点赞 / 收藏 / 关注 / 评论 / 回复
- 评论/回复素材（如有）：原评论要点、用户希望的语气（克制/热情/专业）、是否需要引导提问

若用户的需求是“先搜索某人/某话题的帖子再互动（例如：回复前五条最新帖子评论）”，先用下面的最小核验做筛选定位：

- 优先用回车键（Enter）触发搜索
- 逐条打开候选笔记，做快速相关性核验：标题/正文首屏/话题/配图是否与目标人物或主题一致；明显引流或无关的直接跳过
- 确认相关后再进入评论区执行评论/回复动作

## 工作流

为避免对话过长或批量动作导致失败，默认启用规模限制：

- 批量回复：单次最多处理 10 条可确定的评论，其余分批

### 1) 点赞/收藏（回访标记与运营）

- 先分析“对什么内容点赞/收藏”：
  - 若用户指向具体笔记：进入该笔记页再操作
  - 若用户只说“把刚才那篇/第一篇/这篇收藏”：结合最近打开的笔记、标题、作者与语境选最匹配的一篇；不确定就先停下并要求用户给链接或截图
- 图标识别：
  - 心形：点赞
  - 星形：收藏
- 适用时机：
  - 收集整理阶段：对高价值笔记收藏做“回访标记”
  - 发布后运营：对同主题内容适度互动，提升账号活跃与同赛道链接
- 执行与回传：
  - 动作类型（点赞/收藏）+ 目标链接/标题 + 执行结果 + 失败原因（如有）

### 2) 关注（需要进入主页）

- 先分析“关注谁”：作者本人/评论者/指定账号
- 操作路径：
  - 点击用户头像进入主页 → 点击关注
- 执行与回传：
  - 目标账号（昵称/主页链接若可得）+ 关注结果 + 失败原因（如有）

### 3) 评论（针对帖子本身）

- 先分析“对什么帖子评论”：必须定位到具体笔记
- 评论排序说明（PC 端限制）：
  - PC 端通常无法按“时间顺序/最新评论”对帖子评论进行显式排序，不要去找“筛选/排序/最新”按钮
  - 若用户要求“回最新评论”：只能基于当前页面可见顺序与可见的时间提示（如“刚刚/xx分钟前/xx小时前”）进行识别与优先级处理
- 操作路径（二选一，以页面可用入口为准）：
  - 进入帖子 → 找到“说点什么...”输入框 → 输入评论 → 点击发送
  - 或点击转发分享按钮左侧的大聊天框形状按钮 → 输入评论 → 发送
- 执行与回传：
  - 目标笔记标题/链接 + 评论内容摘要 + 是否已发送/需手动 + 失败原因（如有）

### 4) 回复（评论级）

- 关键区分（防止误把回复当评论）：
  - “评论”是对帖子本身发言：入口是帖子底部“说点什么...”大输入框
  - “回复”是对某一条评论发言：入口是每条评论下的“小聊天框”按钮
  - 回复只允许使用“小聊天框”入口，不允许点击“说点什么...”大输入框
- 单条回复：
  - 点该评论下“小聊天框”按钮进入回复 → 确认不是底部“说点什么...”输入框 → 输入 → 发送
  - 发送后看到该评论下出现回复内容/已回复迹象才算成功
- 批量回复未回复评论：
  - 优先用回复数快速判定：小聊天框旁回复数为 0/无数字 → 视为未回复；大于 0 → 跳过
  - 从上到下扫描可见评论，逐条点“小聊天框”进入回复并发送
  - 若用户要求“最新评论”：优先处理页面上带“刚刚/xx分钟前/xx小时前”的评论（仅能识别，不能排序）
  - 任意一次发现输入框跑到“说点什么...”或发送后没有落在该评论下：立即停止批量操作并说明原因
- 回传：
  - 单条：目标评论要点 + 回复文案 + 是否已发送/需手动
  - 批量：回复文案 + 已回复条数 + 跳过条数 + 失败原因（如有）

## 当前不支持

- 转发与复制链接：当前 GUI 能力不可用，遇到此类需求直接说明限制并停止该动作
