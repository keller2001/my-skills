# 产品评测收集整理工作流（小红书内采集 → 高质量筛选 → 直接阅读提取 → 文档整合）

目标：围绕关键词在小红书平台内收集“值得种草”的高质量内容，给出可解释的价值判断，并把内容整理成可直接粘贴到文档的结构化材料。

## 输入信息（缺什么先补什么）

- 产品对象：品类/型号/价位/使用场景（例如：500 元送妈妈礼物、某型号手机、某款防晒）
- 人群与约束：预算、肤质/体质、使用场景、偏好（“实用/耐用/轻便”等作为筛选维度）
- 目标：想“对标爆款结构”、想“筛选真实体验”、想“沉淀选购清单与购买建议”
- 输出偏好：要点为主（默认）/表格对比（可选）

## 采集原则

- 必须在小红书平台内完成检索与采集（Web/客户端均可），不要用通用搜索替代
- 优先采集“信息密度高、可复用要素多、表达清晰、可验证”的笔记
- 不管用户是否明确提出“高赞/高评/高收藏”，都必须先按高质量逻辑筛选，再做整理归纳，除非提出想要“少赞/低热度/ 少评论”等需求
- 采集时同时记录：链接、作者、发布时间（若可见）、互动信号（若可见）、关键词命中点、可复用的结构与表达
- 最终交付默认不展示互动数字与粉丝数字（除非用户明确要求），但筛选与判断过程必须参考这些信号
- 输出给用户的参考来源时，必须优先提供与用户问题直接相关的笔记条目链接，不要只给搜索页或作者主页链接
- 涉及关键结论、对比判断、参数/价格/功效等事实点时，必须附对应来源链接，不得故意不提供或无说明省略

## 工作流

为避免对比候选过多导致超时或对话过长，默认启用规模限制：

- 搜索轮次：最多 3 轮关键词搜索；仍找不到高质量候选就停止并说明原因
- 候选对比：每轮最多打开 8 篇候选快速扫读（优先看收藏/评论等互动信号与内容密度）
- 最终入选：默认输出 5 篇笔记以内（除非用户明确要更多，改为分批交付）
- 单篇输出上限：摘要最多 2 句；可复用要点最多 5 条；图片信息最多 3 条

### 1) 在小红书内检索与扩词

- 先用“核心词”搜，再叠加限定词缩小范围
- 执行搜索时优先用回车键触发（Enter），不要优先去点击搜索框旁的放大镜按钮
- 检索词生成规则（避免生硬堆叠）：
  - 先把用户需求拆成：核心对象 + 场景/节日 + 预算/价位 + 人群（可选）
  - 把“实用/好用/高级/有质感/不踩雷”等作为筛选维度，不直接塞进搜索词（除非用户明确要求把这个词当关键词搜）
  - 每次搜索用 1-2 个短语即可，优先用自然短语而非四五个词硬拼，不要用逗号/换行把词硬隔开
  - 示例：用户说“帮我找三八节适合送母亲的礼物，五百块左右预算，最好实用一点”
    - 搜索短语 A：三八节 送妈妈 礼物 500
    - 搜索短语 B：送妈妈 礼物 500元
    - 搜索短语 C：母亲节 礼物 500（节日同义扩展）
    - “实用”放到阅读筛选里判断：是否可落地、是否有具体清单/使用场景/避坑点
- 搜索策略指南（小红书 collect 专用）：
  - 平台选择原则：
    - 默认只在小红书内检索与整理（本工作流的唯一主平台）
    - 若用户明确要求“商品价格/购买渠道/比价”：在完成小红书内容整理后，可建议用户再去价格平台验证；不要用比价平台替代小红书内容采集
  - 关键词策略：
    - 先用简洁关键词拿到更多结果，再逐步增加限定词缩小范围
    - 预算优先用数字表达（如 500/500元），人群可用“送妈妈/送长辈/通勤/学生党”等
  - 结果筛选：
    - 通过浏览与滚动对比多篇候选，优先选择更相关、更权威/更可复用的内容
  - 信息验证（在小红书内完成交叉验证）：
    - 对关键结论至少用 2-3 篇不同作者笔记互相印证
    - 若涉及参数/价格/功效等敏感事实，以“多篇一致 + 评论区反馈”作为可信度增强信号
  - 问题处理：
    - 搜索结果不准：换一种说法/同义词/更换人群与场景限定词，或缩短关键词重搜
    - 平台需要登录/验证码：请用户先完成登录/验证后再继续执行检索与筛选
    - 页面操作卡住：刷新、返回重进、换入口（搜索页/话题页/综合/最新），或换短语再搜
- 扩词方向：
  - 评测类：对比、测评、避雷、真实体验、开箱、通勤、学生党、平替、敏感肌、油皮干皮、续航、屏幕、手感
- 同一主题至少覆盖：高互动、长文信息密度、不同人群视角三类笔记

### 2) 候选笔记筛选（先排雷再筛选）

- 剔除：明显硬广、信息空洞、只有情绪无细节、标题党但无支撑
- 优先：有对比、有过程、有参数/清单/步骤、有坑点总结、有图文证据链
- 硬性门槛（必须执行，避免只因标题匹配误选低质量）：
  - 不允许只打开第一篇就定稿；必须打开并快速扫读至少 5-8 篇候选后再选（默认 6 篇）
  - 先用互动信号做粗筛，再进入“直接阅读提取”：优先收藏，其次评论，再次点赞
  - 若互动信号可见且明显过低（例如点赞仅个位数/十几），默认跳过并继续找更高质量候选（除非用户明确要低热度/小众/少赞）
  - 若互动信号不可见：用“内容信号 + 图片信号 + 作者信号”替代互动粗筛，但仍要对比至少 5 篇再选
- 高质量信号（用于筛选，不强制写进最终交付）：
  - 帖子信号：收藏/评论/点赞表现是否显著、互动是否与内容“干货程度”匹配
  - 作者信号：是否垂直领域、过往内容一致性、表达是否稳定可靠
  - 内容信号：是否给出可复用的方法、清单、步骤、对比维度；是否给出明确边界与适用人群
  - 图片信号：是否能“看图复现”（步骤/对比/机位/路线/清单/标注）
- 避免“标题命中但质量很低”的误选：
  - 先看高质量信号再决定是否进入阅读提取：若互动明显过低且无强内容证据链，默认跳过（除非用户明确要低热度/小众）
  - 不只取搜索结果第一篇；至少打开并对比 5-10 篇候选，再选出最符合用户需求且质量更高的若干篇
  - 若只能找到低互动内容：扩大关键词、换同义词/人群词/场景词继续搜，直到找到质量更高的候选为止

### 3) 高价值判断清单（不打分）

目标：让模型学会什么是高价值帖子，并把这些判断用于筛选，而不是为了输出一个分数。

- 内容是否“可复用”：有清单/步骤/对比维度/路线安排/预算拆解/避坑点，能直接抄成用户决策建议
- 图片是否“能看图复现”：关键步骤/对比/机位/路线/价格/标注清晰，图与文互相补强
- 结论是否“有边界”：明确适用人群/前提条件/不适合的人，避免空泛
- 互动是否“匹配干货”：收藏与评论能反映实用性；互动过低且无强证据链的默认跳过（除非用户要低热度）
- 作者是否“可信”：领域垂直、表达稳定、长期输出一致

强制要求：最终整理稿里至少写一句“已按高质量信号（互动/作者/内容/图片）对比多篇候选后选出”，避免只按标题匹配选第一篇。

### 4) 直接阅读提取

目标是基于你在页面上“看得到”的信息，直接提炼出可用于整理的要点，并记录链接与图片承载的信息。

- 展开全文（若有“展开/全文”），并按段阅读正文
- 从正文里提取：
  - 结论句（作者核心观点）
  - 证据/方法（对比维度、步骤、清单、路线、参数、注意事项）
  - 坑点/避雷（明确场景与边界）
- 从图片里提取（按张编号）：
  - 图1-图N分别提供了什么信息（对比图/步骤图/机位图/价格牌/菜单/路线图等）
- 记录“可回访信息”：
  - 作者/发布时间/点赞收藏评论（若可见）
  - 与用户需求的匹配点（为什么被选中）
- 记录当前帖子链接（必须严格按以下步骤）：
  1. 在当前打开的图文帖子页面，将鼠标移动到右下角转发 icon；当光标变为“链接图标”后点击，执行复制链接
  2. 从 GUI 工具返回结果中读取 `clipboard_content`，该字段即当前帖子的链接；将该链接呈现给用户
  3. 注意：不能直接复制浏览器地址栏网址作为帖子链接，必须严格按上述 1、2 步执行
  4. 若页面限制导致暂时无法拿到单条帖子链接，必须向用户说明具体阻塞原因，并提供可回溯替代入口（搜索结果页/作者主页）与后续补链计划

### 4.5) 可选：点赞/收藏作为回访标记（需授权）

- 目的：把“高价值候选笔记”标记下来，便于后续回看与对标，不代替文档化整理
- 规则：
  - 只有在用户明确授权后才执行点赞/收藏
  - 只对“确定的目标链接/当前页面可见笔记”操作，不要批量随机操作
  - 节奏控制与结果回传要求参考 [互动运营工作流](interact.md)

### 5) 整合成文档（建议结构）

- 最终交付：输出一份可直接保存为 `.md` 的“帖子内容整理稿”（你可以直接复制粘贴保存；不要求先创建空文件）
- 文档只要围绕用户需求，把同主题的优质笔记按“评测/旅游攻略/其他类别”整理出来即可，不需要复杂评分表

建议用下面这个最简模板输出（直接复制就是最终交付的 md）：

```md
# {类别}｜{用户需求一句话总结}

## 需求与边界
- 用户要解决的问题：
- 约束（预算/肤质/城市/天数/机型/季节等）：
- 关键词组合（小红书内搜索用）：

## 精选笔记整理（按用户需求筛选）

### 1) {子主题/角度A}
1. **{笔记标题}**
   - 摘要：{这篇讲了什么，1-2句}
   - 可复用要点：{清单/步骤/对比维度/适用人群/购买建议等}
   - 图片信息：{图1-图N分别承载什么信息，用于复用或对标}

### 2) {子主题/角度B}
1. **{笔记标题}**
   - 摘要：
   - 可复用要点：
   - 图片信息：

## 汇总结论（给用户的决策与下一步）
- 关键结论：
- 建议下一步（继续搜哪些词/需要补哪些信息/是否进入创作发布）：
```

## 场景要点

### 产品评测（3C）

- 必收：对比维度、参数/续航/屏幕/拍照/手感、使用场景、真实缺点、购买建议与人群分层
- 图片重点：对比图、细节特写、场景图、参数截图、前后对比

### 产品评测（美妆）

- 必收：肤质/肤况前提、质地与使用手法、妆效/持妆、过敏与避雷（避免医疗化表述）、同价位对比
- 图片重点：上脸近景、光线一致对比、步骤图、色号试色、成分/使用量标注

### 送礼类（预算导向）

- 必收：送礼场景（妈妈/长辈/同事）、实用性证据（使用频率/耐用性/售后）、踩雷点（智商税/不适合人群）
- 图片重点：实物细节、尺寸/容量/材质标注、使用场景图、清单对比图
